Apa itu versioning

Versioning adalah pengaturan versi atau pelacakkan perubahan dari source code. Version Control System (VCS) adalah sistem atau aplikasi untuk melakukan versioning. Versioning memiliki beberapa tujuan antara lain :

  1. Melacak kesalahan di kode program
  2. Mengetahui perubahan yang dilakukan dan siapa yang melakukan perubahan tersebut
  3. Membantu manajemen projek

Apa itu Git dan Github

Salah satu VCS adalah Git yang diciptakan oleh Linus Torvald. Git adalah salah satu VCS populer yang digunakan para developer untuk mengembangkan software secara bersama-sama. Lalu ada Github yang merupakan tempat hosting dari layanan Git.

Cara menggunakan layanan Github

  1. Membuat remote server

    2. git init

    git remote add origin "https..."

  2. Melakukan perubahan di server git

    3. git commit -m "pesan commit disini"

    git remote add origin "https..."

    git push origin master

  3. Membuat branch dan berpindah ke sebuah branch

    4. git branch nama_branch

    git checkout nama_branch

Apa itu UX

UX (User Experience) adalah apa saja yang pengguna rasakan ketika menggunakan sebuah produk. Terdapat 6 tahapan UX design thinking:

  1. Empathize: Melihat dari sudut pandang user
  2. Define: Menggabungkan informasi yang didapatkan saat tahap empathize
  3. Ideate: Alur penggunaan produk
  4. Prototype
  5. Validate
Tahap Prototype dan validate akan sering diulangi untuk mendapat UX yang terbaik

Apa itu UI

UI (User interface) adalah visual yang digunakan user untuk berinteraksi dengan produk. Terdapat 4 pilar UI :

  1. Consistency
  2. Keep the interface simple
  3. Good typography
  4. Offer informative feedback

Apa itu Prototype

Prototype adalah simulasi atau versi sampel dari produk akhir yang bertujuan untuk mendapatkan feedback dari user. Terdapat 3 pilar prototype:

  1. Representation
  2. Precision
  3. Interactivity

Apa itu HTML

Proses web development dibagi menjadi frontend dan backend. Frontend adalah sebuah bagian dari website yang membuat tampilan menarik kepada user. Frontend secara umum tersusun atas 3 komponen:

  • HTML
  • CSS
  • JS
HTML (HyperText Markup Language) adalah sebuah standar yang digunakan secara luas untuk menampilkan halaman web. HTML punya beberapa kegunaan yaitu:
  • Membuat struktur dari halaman website
  • Mengatur tampilan dan isi dari halaman web
  • Membuat beberapa konten untuk halaman web seperti tabel, form, gambar dll

Bagaimana membuat file HTML

Pembuatan HTML bisa menggunakan code/text editor seperti:

  • VS Studio Code
  • Sublime
  • Atom
Sedangkan untuk menjalankan file HTML yang sudah dibuat bisa dengan menggunakan web browser(Chrome, Edge, Firefox, dll)

Tag-tag di HTML

  1. Ada beberapa dokumentasi untuk tag utama HTML:

    2. DOCTYPE html: Mendefinisikan dokumen sebagai HTML5

    html: Elemen root dari halaman HTML

    head: Berisi informasi meta suatu dokumen

    title: Judul dari dokumen

    body: Berisi konten yang akan dimuat di halaman website

  2. Ada beberapa tag yang sering dipakai yaitu:

    3. div: Membungkus elemen HTML

    h1 -> h6: Heading

    p: Paragraf/teks

    br: Baris baru

    a: Hyperlink

    img: Gambar

  3. Untuk membuat sebuah list ada beberapa tag yaitu:

    4. ol: Ordered list/list terurut

    ul: Unordered list/list tak terurut

    li: Item dalam list

  4. Untuk membangun sebuah tabel ada beberapa tag yaitu:

    5. table: Membuat tabel

    tr: Baris tabel

    td: Kolom tabel

    th: Header kolom

  5. Untuk membuat sebuah form ada beberapa tag yaitu:

    6. form: Membuat form

    input: Masukan untuk form

    label: Memberi nama untuk input

Apa itu CSS

CSS (Cascading Style Sheet) yang berfungsi memperindah halaman web. Jika dianalogikan dengan tubuh manusia, HTML yang sebelumnya merupakan kerangka tulang, maka CSS adalah kulit dan pakaian yang membungkus badan sehingga lebih bagus. CSS berguna untuk menghias halaman web dan juga mampu mengatur posisi dan tata letak komponen pada halaman web. Ada 3 cara menambahkan file CSS ke dalam file HTML yaitu:

  • External CSS
  • Internal CSS
  • Inline CSS

Syntax CSS

Syntax CSS terdiri atas selector lalu diikuti kurung kurawal. Di dalam kurung kurawal tersebut ada deklarasi dari CSS. Deklarasi tersebut terbagi menjadi property dan value.

div { background-color: green; width: 200px;}

Selector adalah pola yang digunakan untuk memilih elemen yang akan diberikan style. Ada 3 cara untuk melakukan CSS selector:
  • ID (#)
  • Class (.)
  • HTML tag (Nama_tag)
Untuk atribut HTML, ID bersifat unik sehingga dalam satu file HTML hanya boleh ada satu ID, sedangkan untuk class yang sama bisa dimiliki oleh beberapa elemen dan juga sebuah elemen bisa memiliki lebih dari satu class.

CSS grouping adalah ketika beberapa selector dikelompokkan dalam satu deklarasi style. Caranya adalah dengan menulis kedua selector tersebut yang dipisahkan dengan tanda koma.

h1, h2 { color: green; font-size: 100px;}

Property di CSS

  1. Font

    2. CSS fonts memiliki beberapa properti antara lain:

    • Font: Shorthand property
    • Font-family: Kelompok font teks
    • Font-size: Ukuran font teks
    • Font-weight : Ketebalan font teks
    • Font-style : Style dari font teks seperti italic dll.
  2. Margin & Padding

    3. CSS margin dan padding berfungsi untuk mengatur ruang atau jarak antar elemen. Margin adalah jarak dari border elemen ke arah luar sedangkan padding adalah jarak dari border elemen ke arah konten di dalam elemen tersebut. Margin dan padding memiliki properti yang sama. Jika value dari margin/padding hanya satu maka akan value tersebut akan diterapkan ke seluruh sisi. Untuk mengatur masing-masing sisi bisa dengan menggunakan margin/padding - top/left/bottom/right.

  3. Background

    4. CSS background digunakan untuk mengatur background dari elemen. Ada beberapa properti background antara lain:

    • Background-color: Warna background
    • Background-image : Gambar background
    • Background-repeat : Pengulangan background
    • Background-size : Ukuran background
    • Background-position : Posisi background
  4. Link event

    5. CSS link event (Pseudo classes) adalah style yang akan diterapkan ketika suatu event pada sebuah elemen terjadi. Ada 3 macam event yaitu:

    • hover: Ketika mouse berada di atas elemen
    • active : Ketika elemen di klik
    • visited: Ketika elemen/link sudah pernah di klik/dikunjungi sebelumnya
  5. Display

    6. CSS display untuk menentukan bagaimana tampilan dari suatu elemen. Ada beberapa value dari display yaitu:

    • Block: Elemen dimulai pada baris baru (dari kiri ke kanan)
    • Inline-block: Elemen menggunakan panjang/lebar sesuai yang diperlukan
    • None: Menyembunyikan elemen
  6. Table

    7. CSS table digunakan untuk mengatur style tabel. Border umunya menggunakan shorthand property yaitu border-width, border-style, border-color sesuai urutan tersebut.

Apa itu Bootstrap

Frontend Framework adalah sekumpulan aturan kode yang bisa digunakan untuk mempermudah dalam membuat suatu tampilan website, salah satunya bootstrap. Bootstrap adalah salah satu framework CSS yang populer untuk mengembangkan website yang responsif. Hanya dengan meletakkan kelas-kelas dari Bootstrap pada elemen HTML maka proses layout dan pembuatan komponen jadi lebih mudah.

Kelebihan Bootstrap

  • Gratis
  • Mudah dipelajari
  • Cepat
  • Mendukung responsif

Apa itu Bootstrap

Ada dua cara untuk menggunakan Bootstrap di file kita:

  1. Download file Bootstrap File bootstrap tersebut bisa didownload lalu dilettakan di folder project sehingga kelas-kelas Bootstrap bisa digunakan.
  2. Menggunakan CDN Bisa dengan menggunakan link CDN untuk menghubungkan file HTML dengan Bootstrap sehingga dapat menggunakan kelas-kelasnya.
Ada beberapa kekurangan dengan cara di atas, jika file Bootstrap didownload maka akan lumayan membuat project jadi berat. Sedangkan jika menggunakan CDN(link), kita tidak perlu mendownload tetapi untuk menggunakan Bootstrapnya maka diperlukan koneksi internet.

Apa itu Javascript

Javascript(JS) adalah bahasa pemrograman high level, scripting, untyped dan interpreted, artinya:

  • High level: Bahasanya mirip dengan bahasa manusia daripada mesin
  • Scripting: JS dibuat untuk dapat berinteraksi dengan browser
  • Untyped: JS tidak berpengaruh terhadap tipe data
  • Interpreted : Tak perlu dicompile
Declaration adalah proses pembuatan variabel untuk menyimpan data. Ada 3 cara untuk mendeklarasikan sebuah variabel yaitu let, var dan const. Const harus diberi nilai saat dideklarasikan. Berikut adalah perbedaan antara let, var dan const:
Declaration Redeclaration Reassignment Hoisting Global scope Function scope Block scope
Var O O O O O X O
Let O X O X O X X
Const X X X X O X X

Tipe Data, Method, dan Control Flow

Tipe data di JS dapat dikategorikan menjadi primitives dan object. Primitives adalah pemrosesan terkecil dan elemen paling sederhana seperti string, boolean, number, undefined, null dll. Object adalah unit yang menyimpan properti dan fungsi (method) seperti object, array, function, date, set, map dll.


Method merupakan sebuah fungsi yang terkait dengan object, membuat programnya sesederhana mungkin sesuai kegunaan masing-masing. Bisa diumpamakan method merupakan sebuah shortcut untuk mempersingkat sebuah kode. Macam-macam method di JS antara lain concat, map, foreach, slice, filter, reduce.


Flow di JS ada dua yaitu normal flow dan control flow. Normal flow berarti pengeksekusian statement dari atas ke bawah atau kiri ke kanan yang dilakukan secara berurutan. Control flow berarti mengatur alur eksekusi pada statement atau jalannya program sesuai keinginan kita. Untuk melakukan control flow dibutuhkan dua hal yaitu pengulangan (for, while, do while) dan pengkondisian (if…else, switch, try…catch, break, continue, throw).

Class, Async Wait, dan DOM

Class adalah prototype dari suatu object yang akan dibuat. Method adalah sebuah fungsi yang berada di dalam kelas. atribut adalah kumpulan variabel yang membentuk objek yang dimiliki suatu class. Jika diumpamakan, class ini bertindak sebagai sebuah cetakan untuk membuat sebuah objek baru.


Asynchronous adalah hasil eksekusi atau output tidak selalu berdasarkan urutan kode, tetapi berdasarkan waktu proses. Asynchronous function adalah sebuah fungsi yang bekerja secara asynchronous yang menghasilkan promise sebagai return value nya, tetapi cara penulisan kodenya menggunakan penulisan yang synchronous (standar). Await adalah sebuah keyword yang terdapat pada async function yang tujuannya menghentikan eksekusi sambil menunggu promise nya selesai


Document Object Model (DOM) adalah API untuk HTML yang merepresentasikan halaman web pada suatu dokumen menjadi sebuah object.
Dom selection method:

  • getElementById() : Element
  • getElementByTagName() : HTML collection
  • getElementByClassName() : HTML collection
  • querySelector() : Element
  • querySelectorAll() : nodeList
Dom manipulation:
  • element.innerHTML : Merubah isi tag
  • element.style.propertyCSS : Merubah style dari tag
  • element.setAttribute() : Mengelola/memanipulasi atribut
  • element.classList.add() : Mengelola/memanipulasi class
Dom event:
  • element.innerHTML : Merubah isi tag
  • element.style.propertyCSS : Merubah style dari tag
  • element.setAttribute() : Mengelola/memanipulasi atribut
  • element.classList.add() : Mengelola/memanipulasi class

Apa itu Clean Code

Clean code adalah istilah untuk kode yang mudah dibaca, dipahami dan diubah oleh programmer. Menurut Brendon Gregory: "kode yang bekerja tidak selamanya merupakan kode yang bagus, kodemu juga harus mudah untuk dibaca, dimengerti dan diubah". Menurut Martin Fowler: "Siapapun dapat menulis kode yang dapat dimengerti komputer, programer yang bagus dapat menulis kode yang dimengerti manusia"

Mengapa harus menerapkan Clean Code

  • Kerja bersama dengan orang lain
  • Mempermudah pengembangan fitur
  • Mempercepat proses pengembangan

Karakteristik Clean Code

  • Mudah dipahami
  • Mudah dieja dan dicari
  • Singkat namun deskriptif
  • Konsisten
  • Menghindari penambahan konteks yang tidak perlu
  • Komentar seperlunya
  • Pembuatan fungsi yang bagus
  • Menggunakan konvensi
  • Formatting yang tepat

Jika kode merupakan Clean Code maka pasti akan memenuhi prinsip Clean Code. Prinsip clean code ada dua yaitu KISS (Keep It So SImple) yang artinya membuat fungsi khusus untuk melakukan satu hal saja. Prinsip yang satunya adalah DRY (Don't Repeat Yourself) yang artinya hindari menduplikasi kode.
